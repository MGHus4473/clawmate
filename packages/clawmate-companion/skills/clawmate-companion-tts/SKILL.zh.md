# ClawMate Companion TTS

当某条回复更适合用“短语音条”表达，而不是直接发可见文字时，使用这个 skill。

## 什么时候用

优先在这些场景使用：

- **必须发语音**
  - 用户明确要求：`给我发语音` / `读给我听` / `你讲给我听`
  - 晚安陪伴、哄睡、讲故事、念情话
  - 用户明显在寻求安慰：委屈、孤独、焦虑、崩溃、想哭

- **适合发语音**
  - 早安、晚安、下班问候
  - 节日祝福
  - 鼓励
  - 庆祝
  - 想念、撒娇、亲密互动

## 什么时候不要用

以下内容不要转成语音：

- 工具型问答
- 知识解释
- 长列表
- 代码
- 设置说明
- 太长、不适合自然口播的内容
- 用户刚表达“现在不方便听”“先别发语音”

## 工作流

1. 先判断这条回复是否应该走语音。
2. 如果不该发语音，就正常发文字，不调用工具。
3. 如果该发语音，先写一段适合口播的短文本，再调用：

```ts
clawmate_generate_tts({
  text: "<适合口播的短文本>"
})
```

## 口播文本规则

- `text` 是给 TTS 的，不是给用户看的
- 要短、自然、亲密，像真实女友发短语音
- 一般控制在 1-3 句话
- 更口语化，不要像写文章
- 不要用 Markdown
- 不要写列表
- 不要写代码
- 不要把长说明整段念出来

## 工具结果处理

### 成功时

1. 不要再发送同内容可见文字
2. 工具结果会返回本地音频路径 `audioPath`
3. 使用这个路径对应的音频，把语音发给用户

### 失败时

1. 使用返回的 `message` 自然降级成文字回复
2. 不要伪造音频路径或发送协议


## 禁止行为

- 不要为代码、教程、长说明调用 TTS
- 已经发语音时，不要重复发送同内容文字
- 不要编造本地文件路径
- 不要把所有回复都变成语音
